# Multi-Agent Research Assistant

A multi-agent system that researches a topic, summarizes the findings, drafts a structured report, and pauses for human approval before finalizing. Specialized agents are built with Microsoft AutoGen, the overall workflow is orchestrated as an event-driven state machine with LangGraph, and FAISS backs both the research knowledge base and the agent's persistent long-term memory.

## Tech Stack

Python, LangGraph, Microsoft AutoGen (autogen-agentchat), OpenAI API, FAISS

## Dataset

`data/knowledge_base/` contains 15 short reference articles on distinct research-relevant topics (AI, quantum computing, biotechnology, climate policy, cybersecurity, space exploration, semiconductors, electric vehicles, genomics, blockchain, the renewable energy grid, 5G, robotics, fintech, and remote work trends), generated by `src/generate_knowledge_base.py`. `src/build_knowledge_base.py` chunks and embeds them into a FAISS index under `vector_db/`, giving the researcher agent a real, queryable corpus instead of depending on a live web search service.

## How It Works

`src/agents.py` builds three AutoGen `AssistantAgent`s — researcher, summarizer, writer — each backed by an `OpenAIChatCompletionClient`. The researcher is given three function-calling tools (`src/tools.py`):

- `search_knowledge_base`, which queries the FAISS-indexed corpus in `vector_db/` (built by `src/knowledge_base.py`).
- `record_finding` and `recall_findings`, backed by `src/memory.py`'s `ResearchMemory`, a separate FAISS index plus a JSON metadata sidecar that persists findings to disk. This is the agent's long-term memory: a finding recorded during one research session can be recalled by a later session on a related topic, even in a new process, which is what gives the system persistent agent memory rather than starting from a blank slate every run.

`src/graph.py` wires the agents into a LangGraph `StateGraph`:

1. **research** — the researcher agent recalls relevant past findings, searches the knowledge base, and records new findings.
2. **summarize** — the summarizer agent condenses the findings.
3. **write** — the writer agent drafts a structured report, incorporating any reviewer feedback from a prior loop.
4. **human_review** — a hard checkpoint that calls LangGraph's `interrupt()`, pausing execution and surfacing the draft report for a human to approve or send back with feedback. This is the human-in-the-loop (HITL) step.
5. **finalize** — promotes the approved report to the final output.

After `human_review`, a conditional edge (`route_after_review`) is the event-driven transition: it reads the state produced by the human's decision and either loops back to **write** with their feedback or proceeds to **finalize**, capped by `max_revisions`. `build_graph` accepts any LangGraph checkpointer; `src/pipeline.py` uses `AsyncSqliteSaver`, so the paused state at an `interrupt()` is written to disk under `memory_store/checkpoints.sqlite` and survives even if the process restarts before a human responds.

`src/pipeline.py` drives this from the command line: it invokes the graph, and whenever the result contains `__interrupt__`, it prints the draft report and the instructions, takes the human's response from stdin, and resumes the graph with `Command(resume=response)`.

## Setup

```bash
pip install -r requirements.txt
cp .env.example .env
```

Fill in `OPENAI_API_KEY` in `.env`. It's used both for the agents' chat completions and for the embeddings that power the knowledge base and persistent memory.

## Build the Knowledge Base

```bash
python src/generate_knowledge_base.py
python src/build_knowledge_base.py
```

## Run

```bash
python src/pipeline.py "The state of quantum computing in 2026"
```

The pipeline will research, summarize, and draft a report, then stop and print it for your review. Type `approve` to accept it, or type feedback to send it back for another draft (up to `max_revisions`, which defaults to 2).

## Run with Docker

```bash
docker build -t multi-agent-research-assistant .
docker run -it --env-file .env multi-agent-research-assistant python src/pipeline.py "your topic"
```

## Run Tests

```bash
pytest tests/
```

Tests cover the FAISS-backed knowledge base and persistent memory store (both with a fake embedding function so no OpenAI calls are needed), the tool wrapper formatting logic, the post-review routing logic, and that the graph compiles with the expected nodes. None of them call OpenAI. Running the full pipeline end to end requires a valid `OPENAI_API_KEY`.
